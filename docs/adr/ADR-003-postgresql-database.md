# ADR-003: Выбор PostgreSQL как основной СУБД

## Контекст
Система требует:
- Сложные запросы с JOIN (отчёты, аналитика)
- Транзакционная целостность (согласования)
- Аудит и история изменений
- JSONB для гибкой структуры данных

## Рассматриваемые варианты

### 1. PostgreSQL (Выбран)
- **Плюсы:**
  - ACID транзакции
  - JSONB для полуструктурированных данных
  - Отличная аналитика (aggregations, window functions)
  - Полнотекстовый поиск
  - Материализованные представления
  - Встроенная репликация
- **Минусы:**
  - Сложность шардинга

### 2. MongoDB
- **Плюсы:** Гибкая схема, горизонтальное масштабирование
- **Минусы:** Ограниченные транзакции (сессии), отсутствие JOIN

### 3. MySQL
- **Плюсы:** Простота, популярность
- **Минусы:** Ограниченная аналитика, JSON менее развит

## Решение
Выбран **PostgreSQL 16** благодаря:
- JSONB для контента отчётов
- Сложные агрегации для аналитики
- Аудит через триггеры/Event Sourcing
- Materialized Views для дашбордов
- Row-Level Security (RLS)
